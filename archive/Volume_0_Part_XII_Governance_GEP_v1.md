# Volume 0 - Part XII: Governance, Constitutional Change & GEP

## Constitutional Rules
- Core ontology changes MUST use the GEP process.
- Economic Law definitions MUST remain backward traceable.
- Breaking architectural changes SHOULD include migration tooling.
- Rejected GEPs SHALL remain archived with rationale.
- Every accepted GEP MUST update the Platform Bible where applicable.

## Review Workflow
1. Draft GEP
2. Architecture Review
3. Scientific Council Review
4. Red Team Challenge
5. Historical Replay
6. Public Comment (optional)
7. Final Vote
8. Publication
9. Implementation
10. Post-Implementation Audit
